# 給民眾使用者：你的健康資料安全嗎？

如果你正在猶豫要不要用 NHI-FHIR Bridge，這份文件解釋我們**實際做了什麼**來保護你的健康資料。**不講工程術語**，用大白話。

> 最後更新：2026-05-27（對應 bridge **v0.9.5**）

---

## 一句話結論

**你的資料只會在你自己的電腦上跑、絕對不會傳到我們（開發者）的伺服器、沒有任何 AI / 雲端服務碰得到。**

---

## 1. 資料路徑

```
（健保署）            （你的瀏覽器）          （你選的去處）
健保存摺  ───►  NHI-FHIR Bridge  ───►  下載到你電腦的下載夾
```

- ✅ 開發者（我們）**完全摸不到資料** — 我們沒有任何伺服器在運作
- ✅ Bridge 是「跑在你瀏覽器內」的轉換工具，跟你的健保存摺分頁互通
- ✅ 走到「下載到電腦」就結束了，**檔案就在你電腦的下載夾**

下載完之後你可以選擇用「醫析 MediPrisma」開啟瀏覽（見後段）。

---

## 2. 我們**保證不做**的事

| 不做的事 | 為什麼這對你重要 |
|---------|--------------|
| ❌ **不把資料傳到我們的伺服器** | 開發者根本沒有伺服器在跑 — 你的資料不可能流到我們手上 |
| ❌ **不接 AI / ChatGPT / Gemini** | 整個 bridge 沒有任何 AI 程式碼。轉換邏輯是純粹「規則對應」，跟用 Excel 公式一樣可預測 |
| ❌ **不收集使用統計 (telemetry)** | 我們不知道你裝了、用了幾次、是否成功。**因為我們根本沒寫收集程式** |
| ❌ **不要求註冊帳號、不要求 email** | 沒有 login、沒有 cookie |
| ❌ **不接觸你的健保存摺密碼** | Bridge 不會出現在登入畫面，密碼是你自己在健保署網站打 |
| ❌ **不同步資料到 Google 帳號** | 你填的性別 / 生日只存「瀏覽器本機」，不會跟其他裝置同步 |
| ❌ **不會偷偷下載檔案** | 每次下載**一定彈出「另存新檔」對話框**，由你選位置 + 確認檔名才落地 |

---

## 3. 我們**主動做的**保護

| 防護 | 解決什麼風險 |
|------|--------------|
| 暫存資料在「**瀏覽器階段儲存區**」 | 關掉瀏覽器，所有暫存自動清空 |
| **下載完成的瞬間立刻清掉暫存** | 你存到電腦的瞬間，瀏覽器裡的副本就沒了 |
| **1 小時自動清掃** | 你忘了下載 + 沒關瀏覽器，1 小時後 bridge 也會自動清 |
| **拒絕別的瀏覽器擴充功能跟我們溝通** | 防止你裝的其他擴充功能假冒身份來偷資料 |
| **下載一定跳「另存新檔」對話框** | 你親眼確認儲存位置 + 檔名，沒有「不知不覺檔案出現」 |
| **改姓名 / 性別 / 生日 → 自動清掉前一個人的暫存** | 診間多人共用瀏覽器不會混 |
| **升級時清掉舊版殘留** | 老版本留下的暫存升級時自動掃乾淨 |

## 5. 怎麼用會更放心

| 小提醒 | 說明 |
|--------|------|
| 下載的 JSON 檔案請好好保管 | 那是你的個人健康紀錄，不建議隨便分享、傳給陌生人或存到不信任的雲端 |
| 不要在公共電腦使用 | 網咖、圖書館電腦、借來的筆電 — 都可能被別人看到資料 |
| 保護健保署登入 | Bridge 本身不接觸你的健保密碼，好好保管健保卡 + 註冊密碼即可 |
| 「醫析 MediPrisma」AI 問答功能 | 醫析的「檢驗趨勢」、「用藥時間軸」等視覺化功能都是**本機運算**（檔案在你瀏覽器跑）。但如果你按了「AI 問答」按鈕，會把資料送給 OpenAI / Gemini 等雲端 AI 模型才能產生回答 — **這是雲端 LLM 的本質**，無法純本機完成。不想用 AI 就只用前面那些本機功能即可 |

## 6. 常見疑慮 Q&A

**Q1: 我裝了 bridge，會不會被當成「使用者授權給開發者用」？**
不會。Bridge 是「裝在你電腦上幫你做事」的工具，跟 line 的對話 / 通訊軟體不一樣 — 我們收不到任何東西。

**Q2: 我能不能完全離線使用？**
擴充功能本身可以離線，但你**必須連線才能抓健保存摺**（健保署 API 在雲端）。一旦資料到你瀏覽器，就可以離線了，後續轉換 + 下載都不需要網路。

**Q3: 我擔心健保署看到我用 bridge → 影響我什麼權益**
你登入健保存摺、按那些按鈕，跟你直接用網頁上的功能，健保署看到的 API call 是一模一樣的。Bridge **不會**對健保伺服器留特殊指紋。

**Q4: Bridge 升級到 1.0 / 2.0 後會不會改成收費 / 偷偷加 AI？**
- **開源 + Apache License 2.0** — 任何更新都會留 git commit 紀錄，全世界都看得到。你可以 fork 一份不含新功能的版本繼續用
- 大改動會在 release notes 寫清楚
- 你不喜歡新版可以**不更新**（chrome 商店版會自動更新，但用「下載 zip 自行載入」就不會）

## 7. 還是不放心怎麼辦

完全可以理解。你有 3 個選擇：

1. **不用 bridge，繼續用健保署官網**（功能有限但最安心）
2. **找你信任的工程師朋友幫你 review code** — 1 小時內可以掃完關鍵檔案
3. **先下載到電腦試試**，下載完不去開啟 SMART App，自己看 JSON 內容是否合理。確認沒問題再升級用法

# 附錄：進階模式（本機伺服器 + Dashboard）

> 以下只跟**進階用戶**（自架 Docker backend 搭配 Dashboard / 自己寫 SMART App）有關。**99% 的民眾用「下載到電腦」就好，不需要看這段**。

## 進階模式是什麼

擴充功能裡有個「⚙️ 進階設定 → 啟用本機伺服器模式」開關。打開後，你可以：

- 跑一個本機 Docker backend（FHIR Server），把多次同步累積起來
- 用 Dashboard (`http://localhost:3010`) 瀏覽多個病人
- 一鍵 launch 自架的 SMART App 看資料

**這需要你有基本的開發環境** — 會用 terminal、知道什麼是 Docker、能設定 backend URL。

## 進階模式專屬的安全防護

| 防護 | 解決什麼風險 |
|------|------|
| **Backend 預設只綁 127.0.0.1 (loopback)** | 同網段其他電腦無法存取 |
| **Dashboard 防 CSRF（檢查 Origin）** | 防止你開著 dashboard 時、被其他網頁偷觸發操作 |
| **SMART App URL 必須是 https:// 或本機** | 防止輸入錯誤的 URL 把 launch token 送陌生網站 |
| **FHIR PHI 端點走 SMART OAuth2 + PKCE** | 標準健康資料授權框架 |
| **Patient.id 用 SHA-1 雜湊**（不放原始身分證） | 萬一資料庫被讀，URL 路徑不會直接洩漏身分證 |
| **PHI 寫入端點（/sync, /fhir/import…）需要 `SYNC_API_KEY`** | 額外的存取保護 |

## 想改預設配置時

預設配置（backend 只綁本機 loopback、單人試用）就是安全的。如果你想客製化：

- 想開放 backend 給 LAN 其他電腦用 → 建議設個強的 `SYNC_API_KEY`
- 想多人 / 多病人臨床使用 → 超出這份文件範圍，請看 [ARCHITECTURE.md](https://github.com/voho0000/NHI-FHIR-BRIDGE/blob/main/docs/ARCHITECTURE.md)

詳細技術設計見 [docs/ARCHITECTURE.md](https://github.com/voho0000/NHI-FHIR-BRIDGE/blob/main/docs/ARCHITECTURE.md)。
