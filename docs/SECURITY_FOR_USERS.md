# 給民眾使用者：你的健康資料安全嗎？

如果你正在猶豫要不要用 NHI-FHIR Bridge，這份文件解釋我們**實際做了什麼**來保護你的健康資料。**不講工程術語**，用大白話。

> 最後更新：2026-05-27（對應 bridge **v0.9.3**）

---

## 一句話結論

**你的資料只會在你自己的電腦上跑、絕對不會傳到我們（開發者）的伺服器、沒有任何 AI / 雲端服務碰得到。**

如果你對「另一個 SMART app 是不是也這樣？」有疑慮，這份文件也會講清楚邊界。

---

## 1. 資料路徑：你的資料會去哪裡？

```
（健保署）             （你的瀏覽器）           （你選的去處）
健保存摺  ───►  NHI-FHIR Bridge  ───►  （3 選 1）
                                          ① 下載到你電腦下載夾
                                          ② 自架的本機伺服器（進階）
                                          ③ 開啟 SMART App 給你自己看
```

**重點**：
- ✅ 開發者（我們）**完全摸不到資料** — 我們沒有任何伺服器在運作
- ✅ Bridge 只是「你瀏覽器裡的轉換工具」，跟你的健保存摺 tab 在同一個瀏覽器內互通
- ✅ 走到「下載到電腦」就結束了，資料就在你電腦的下載夾，不會再傳給任何人

---

## 2. 我們**保證不做**的事

| 不做的事 | 為什麼這對你重要 |
|---------|--------------|
| ❌ **不把資料傳到我們的伺服器** | 開發者根本沒有伺服器在跑 — 你的資料不可能流到我們手上 |
| ❌ **不接 AI / ChatGPT / Gemini** | 整個 bridge 沒有任何 AI / LLM 程式碼。轉換邏輯是純粹「規則對應」，跟用 Excel 公式一樣可預測 |
| ❌ **不收集使用統計 (telemetry)** | 我們不知道你裝了、用了幾次、是否成功。**因為我們根本沒寫收集程式** |
| ❌ **不要求註冊帳號、不要求 email** | 沒有 login、沒有 cookie、沒有「給我們你的聯絡資料」 |
| ❌ **不接觸你的健保存摺密碼** | Bridge 不會出現在登入畫面，密碼是你自己在健保署網站打 |
| ❌ **不同步資料到 Google 帳號** | 你在 popup 填的性別 / 出生年只存「瀏覽器本機」(`chrome.storage.local`)，不會跟其他裝置同步 |
| ❌ **不會偷偷下載檔案** | 每次下載**一定彈出「另存新檔」對話框**，由你選位置 + 確認檔名後才落地 |

## 3. 我們**主動做的**保護（v0.9.3 為止）

| # | 防護 | 解決什麼風險 |
|---|------|--------------|
| 1 | **資料只在「瀏覽器階段儲存區」(`chrome.storage.session`)** | 關掉瀏覽器，所有暫存自動清空 |
| 2 | **下載完成的瞬間立刻清掉暫存** | 你存到電腦的瞬間，瀏覽器裡的副本就沒了 |
| 3 | **1 小時 TTL 自動清掃** | 你忘了下載 + 沒關瀏覽器，1 小時後 bridge 也會自動清 |
| 4 | **拒絕別的瀏覽器擴充功能跟我們溝通** (`sender.id` check) | 防止你裝的其他擴充功能假冒身份來偷資料 |
| 5 | **「另存新檔」對話框** (`saveAs: true`) | 你親眼確認儲存位置 + 檔名，沒有「不知不覺檔案出現」 |
| 6 | **改姓名/性別/生日視為換人 → 自動清掉前一個人的暫存** | 診間多人共用瀏覽器不會混 |
| 7 | **debug 用的原始 NHI 回應 stash 預設關閉** | 不會留下醫療原始資料的「除錯快照」 |
| 8 | **升級時清掉舊版殘留** | v0.8.7 之前的暫存升上來會被掃乾淨 |
| 9 | **SMART App URL 必須是 https:// 或本機** *(v0.9.3+)* | 防止輸入錯誤的 URL 把 launch token 送到陌生網站 |
| 10 | **Dashboard 防 CSRF** *(v0.9.3+)* | 防止你在其他網頁開著 dashboard 時被偷觸發操作 |
| 11 | **檔名帶 bridge 版本號** *(v0.9.2+)* | 例：`nhi-P10109XXXX-...-v0.9.3.json` — 之後出問題能快速判斷 |
| 12 | **發布版本不含 source map** *(v0.9.3+)* | 壓縮 .crx 大小，避免內部 dev 註解外流 |

## 5. ⚠️ 你需要自己負責的事

Bridge 端做了上面那些防護，但**有些事我們替不了你**：

| 你的責任 | 說明 |
|---------|------|
| **下載的 JSON 檔案請好好保管** | 一旦存到電腦，這個檔案的安全就是你的事。不要隨便分享、不要傳給陌生人、不要存到不信任的雲端 |
| **不要在公共電腦使用** | 網咖、圖書館電腦、別人借你的筆電 — 都可能被別人看到資料 |
| **使用「醫析 MediPrisma」AI 問答功能 = 同意把資料給雲端 AI 供應商** | 那個按鈕一按，資料就會送到 OpenAI / Gemini 等。**這跟 bridge 無關**，是 MediPrisma 自己的設計。不想用 AI 就只用「檢驗趨勢」「用藥時間軸」等本機功能 |
| **保護健保署登入** | Bridge 不接觸你的健保密碼。但如果你的健保帳號被盜，盜用者也能用 bridge 抓你的資料。請保護好健保卡 + 註冊密碼 |
| **了解 Chrome 擴充功能本身的權限模型** | Chrome 擴充功能在你電腦上跑、有你授權的權限。出事的話 Chrome 也無法替你還原資料 |

## 6. 常見疑慮 Q&A

**Q1: 我裝了 bridge，會不會被當成「使用者授權給開發者用」？**
不會。Bridge 是「裝在你電腦上幫你做事」的工具，跟 line 的對話 / 通訊軟體不一樣 — 我們收不到任何東西。

**Q2: 如果開發者突然惡意更新版本，會偷我資料嗎？**
理論上有可能（任何 auto-update 軟體都有此風險）。緩解：
- **整個專案開源** — 任何更新都會留 git commit 紀錄，全世界都看得到
- **獨立安全審查** — 每次大版會跑
- **檔名有版本號** — 出問題能溯源到「是哪版」
- **可以選擇不自動更新** — 從 Chrome Web Store 安裝會自動更新；用「下載 zip 自行載入」就完全不會自動更新

**Q3: 我能不能完全離線使用？**
擴充功能本身可以離線，但你**必須連線才能抓健保存摺**（健保署 API 在雲端）。一旦資料到你瀏覽器，就可以離線了，後續轉換 + 下載都不需要網路。

**Q4: 我擔心健保署看到我用 bridge → 影響我什麼權益**
你登入健保存摺、按那些按鈕，跟你直接用網頁上的功能，健保署看到的 API call 是一模一樣的。Bridge **不會**對健保伺服器留特殊指紋。

**Q5: Bridge 升級到 1.0 / 2.0 後會不會改成收費 / 偷偷加 AI？**
- **開源 + Apache License 2.0** — 即使開發者哪天「轉性」加了，你可以 fork 一份不含那個功能的版本繼續用
- 大改動會在 release notes 寫清楚
- 你不喜歡新版可以**不更新**

## 7. 還是不放心怎麼辦

完全可以理解。你有 3 個選擇：

1. **不用 bridge，繼續用健保署官網**（功能有限但最安心）
2. **找你信任的工程師朋友幫你 review code** — 1 小時內可以掃完關鍵檔案
3. **先用「下載到電腦」模式試試**，下載完不去開啟 SMART App，自己看 JSON 內容是否合理。確認沒問題再升級用法
